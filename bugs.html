Critical — broken functionality
Critical
Inspector tabs (Style / Settings / Actions) have no click handlers
The three inspector tabs at the top of the right panel look interactive but no JavaScript is attached to them. All inspector sections — Position, Text, Style, Behavior, Canvas, Options — are always visible simultaneously regardless of which tab appears active. The Settings and Actions tabs are purely decorative.
<!-- HTML line 190 — no JS event listeners anywhere --> <div class="inspector-tabs"> <button class="inspector-tab active">Style</button> <button class="inspector-tab">Settings</button> <button class="inspector-tab">Actions</button> </div>
Add a click handler mirroring the sidebar tab logic — toggle an active class on tabs and show/hide the relevant inspector section groups, using display:none on the inactive ones.
Critical
Arrow-key element moves are never saved to history or autosaved
The keyboard handler moves selected elements on every arrow keypress and calls render(), but never calls saveHist(). Any layout changes made exclusively with arrow keys will be silently lost on page reload or after the next undo.
// line 334 — missing saveHist() at the end if(['ArrowUp','ArrowDown','ArrowLeft','ArrowRight'].includes(e.key)){ arr.forEach(el=>{ ... }); render() // ← no saveHist() call }
Append saveHist() after the render() call inside the arrow key handler, or debounce it to avoid a history entry on every keydown repeat.
Critical
Bottom toolbar tool buttons (hand, zoom-pan, frame) are non-functional
The bottom bar has four tool buttons — Select, Hand, Search, and Frame. Only the Select button has any active styling; none of the four have event listeners. Clicking hand or frame does nothing. Users who expect pan or zoom modes will get no feedback.
<!-- line 186 — no onclick or event delegation --> <button class="tool-square active">⌖</button> <!-- select --> <button class="tool-square">✋</button> <!-- hand: nothing --> <button class="tool-square">⌕</button> <!-- zoom: nothing --> <button class="tool-square">⧉</button> <!-- frame: nothing -->
Either wire up the buttons (hand mode = workspace pan, frame mode = add a frame element) or remove the three non-select buttons to avoid a false affordance.
Critical
Exported HTML prototype embeds broken FORGE-internal CSS
betterExportHtml() (line 472) copies the FORGE application's internal stylesheet verbatim into every exported prototype. This includes classes like .qa-table, .guide-section, and .health-chip that reference custom CSS variables (var(--line), var(--muted), var(--green)) which are never defined in the export. The export also contains unrelated UI utility classes that add dead weight.
Strip the <style> block in the export to only the minimal rules needed for prototype layout: body, .page, and absolute positioning. Remove all FORGE-internal component classes entirely.
Critical
Layer panel reorder buttons overflow the 3-column grid layout
The renderLayers override (line 583) appends ↑ and ↓ buttons after the existing 3-item row. The .row-item CSS uses grid-template-columns: auto 1fr auto — only 3 columns are defined. The extra buttons fall into implicit columns 4 and 5, wrapping onto a second row and producing a visually broken stacked layout. Since renderLayers calls originalRenderLayers() first then mutates the fresh DOM, this happens on every single render.
Either widen the grid template to auto 1fr auto auto auto, or merge the reorder logic into originalRenderLayers so the row is built with all 5 columns up front.
Significant — notable functional gaps
Significant
Device select and canvas size select are not synchronized
The toolbar has a #device dropdown and the inspector has a #canvasSize dropdown; both call setDevice(). But neither updates the other. Switching to Mobile via the toolbar leaves the inspector showing "Desktop 1200×800". Changing canvas dimensions in the inspector doesn't reflect back in the toolbar device label either.
At the end of setDevice(v), set document.getElementById('device').value = v and document.getElementById('canvasSize').value = v so both always mirror the current state.
Significant
Selected element icon in the inspector never updates
The inspector's element card shows a #selectedIcon element (line 192) that always renders ▢. syncInspector() updates selectedName and selectedHelp but never touches selectedIcon, so buttons, images, text, toggles, and containers all show the same generic box icon.
Add a type-to-icon map in syncInspector() and set document.getElementById('selectedIcon').textContent = iconFor(el.type).
Significant
Asset base64 data is stored in every history entry — memory bomb
state.assets is included in exportProjectObject(), which is called by pushHistory() on every action. With a history cap of 150 entries, a single uploaded 500 KB image creates up to ~75 MB of repeated base64 data in memory. Multiple images compound this fast enough to exhaust the JS heap and crash the tab.
Exclude assets from history snapshots. Store assets in a separate state.assets map outside the undo stack, and only include asset IDs (not data) in element records.
Significant
No file size limit on image asset uploads
The asset uploader (line 762) accepts any image/* file with no size check. A multi-megabyte photo reads in fine via FileReader, but the resulting base64 data URL is embedded in every autosave, every history snapshot, and every JSON export. One 4 MB photo produces a ~5.3 MB base64 string that will quickly exceed localStorage's 5–10 MB browser quota and silently fail future autosaves.
Add a guard: if(f.size > 1.5 * 1024 * 1024){ alert('Image must be under 1.5 MB'); return; }. Optionally resize client-side with a <canvas> before storing.
Significant
Context menu has no viewport edge detection
openCtx() positions the context menu at e.clientX / e.clientY with no adjustment for the menu's own dimensions. Right-clicking an element near the bottom-right corner of the screen causes the menu to render mostly or entirely off-screen, with no way to access the options.
After positioning, clamp: ctx.style.left = Math.min(e.clientX, window.innerWidth - ctx.offsetWidth - 8) + 'px' and similarly for top.
UX / UI — design and interaction problems
UX
No Escape key handler to deselect or cancel operations
There is no global Escape handler in the window keydown listener. Users cannot press Escape to deselect elements, dismiss an active drag, clear the selection, or exit Preview mode — all standard interactions in design tools. The only Escape handler is local to the inline text editor.
Add if(e.key === 'Escape'){ state.selected = []; if(state.preview) togglePreview(); render(); } to the global keydown listener.
UX
Inspector shows all sections for every element type regardless of relevance
A selected Text element shows Position, Text, Style, Behavior, Canvas, and Options sections simultaneously — including Canvas and Options which are unrelated to the element. A Line element shows a font weight selector. Showing every field for every type adds noise, makes the panel feel unfinished, and buries the most useful properties.
Define a visibleSections map keyed by element type and hide irrelevant sections via inspector-section.hidden in syncInspector(). Canvas and Options should always be visible and unaffected by selection.
UX
Nesting is metadata-only with no visual feedback
nestSelection() sets el.parent = container.id but render() never reads the parent property to offset, clip, or visually group elements. The toast message "Nested metadata saved" is technically honest but completely unintuitive. Users who nest expecting visual containment will see no change at all.
Either implement visual nesting in render() by positioning child elements relative to their parent's bounds, or remove the Nest/Unnest UI entirely and leave the grouping system (which does work visually) as the only grouping mechanism.
UX
Number inputs trigger a full re-render on every keystroke
All inspector fields bind to the input event, which fires on every character change. Typing "1200" in the X position field triggers 4 re-renders at values 1, 12, 120, and 1200. This causes the canvas to visually jump through intermediate positions and the cursor in the input to behave unexpectedly in some browsers.
Switch the number inputs (X, Y, W, H, font size, radius, opacity) from input to change events. Keep input only on the text content textarea where live feedback is expected.
UX
No active-page indicator in the toolbar or status bar
With multiple pages, there is no persistent display of which page you are currently editing. The only indication is the highlighted row in the sidebar pages panel, which is only visible if the Pages tab is active. Switching tabs hides this context entirely.
Add the current page name to the status bar or as a breadcrumb in the topbar, updated on every render() call via page().name.
UX
Theme cards have no color swatches
The Quick Themes section in the Templates panel shows four cards (Forge, Slate, Light, Terminal) with only text names and subtitles. The .theme-swatch CSS class is fully defined in the stylesheet with height, radius, and hover styles, but no swatch elements exist in the HTML — the class is completely unused.
Add two <div class="theme-swatch"> elements per card, colored with the theme's accent and background values, so users can preview the palette before applying.
UX
ZIP export silently produces an uncompressed archive
The Export Manager describes the ZIP bundle as exporting "a real local ZIP." The custom makeZip() writer (line 780) sets the compression method field to u16(0) — store mode, no compression. A 200 KB HTML file produces a 200 KB ZIP with no size reduction. Users expecting compression will be surprised by the file size.
Update the Export Manager card description to say "uncompressed ZIP archive," or replace the custom writer with the CompressionStream API (supported in modern browsers) for real DEFLATE compression.
Dead code and redundancy
Dead code
Three unused bundle/export functions
exportZipHtmlFiles() (line 448), exportZipLikeBundle() (line 723), and makeFullExportHtml() (line 732) are defined but never called from any button, handler, or other function. They are earlier iterations superseded by exportRealZip() and betterExportHtml().
Delete all three functions. They add ~30 lines of noise and confuse anyone reading the export pipeline.
Dead code
exportBundle() duplicates exportRealZip()
exportBundle() (line 539) downloads a .json file containing stringified HTML and JSON keys — it's a fake "bundle," not a ZIP. It is never wired to a button. Meanwhile exportRealZip() is the function actually called by the Export Manager's ZIP button. The naming overlap creates confusion about which function handles what.
Delete exportBundle() and rename exportRealZip() to exportBundle() or exportZip() for clarity.
Dead code
.theme-swatch CSS class is fully styled but never used in HTML
Lines 72–73 define .theme-swatch with height, border-radius, border, cursor, and hover outline rules. No element in the HTML has this class assigned anywhere — it was likely a stub added during a design pass that was never followed through.
Dead code
exportHtml() is an empty wrapper
Line 332: function exportHtml(){ return betterExportHtml() }. This one-liner indirection serves no purpose — all callers could directly call betterExportHtml(). The name "betterExportHtml" also suggests it's still a work-in-progress replacement rather than the canonical export function.
Rename betterExportHtml to exportHtml, delete the wrapper, and update the two call sites.
Minor and code quality
Minor
document.execCommand('selectAll') is deprecated
The inline text editor (line 520) uses document.execCommand('selectAll', false, null) to pre-select text. This API is deprecated in all modern browsers (removed in Safari, flagged in Chrome/Firefox) and may silently fail.
Replace with: const range = document.createRange(); range.selectNodeContents(ed); const sel = window.getSelection(); sel.removeAllRanges(); sel.addRange(range);
Minor
Side panels and inspector missing Firefox scrollbar styling
Custom scrollbar rules use ::-webkit-scrollbar but not the Firefox standard scrollbar-width / scrollbar-color properties. Only .toolbar-main has scrollbar-width: none. Firefox users see the default OS scrollbar in the component panel, layers panel, and inspector.
Add scrollbar-width: thin; scrollbar-color: #3f5268 #090e15; to .side-panel and .inspector-body.
Minor
Border color and border width cannot be edited from the inspector
Every element is created with a border color (#cbd5e1) and borderWidth: 1. These values are rendered in render() but there are no corresponding inspector input fields mapped in the map object. Users can change fill and text color but have no UI access to the element border.
Minor
History limit of 150 entries with no size guard
pushHistory() caps history at 150 entries with state.history.shift(). But because each entry is a full serialized snapshot (including assets), large projects can consume hundreds of MB before the cap triggers. The cap protects against infinite growth but not against excessively large individual snapshots.
Reduce the cap to 50 and log a console warning when a single snapshot exceeds a size threshold (e.g. 500 KB) to alert developers.
Minor
Roadmap panel lists "ZIP export" as incomplete when it's already shipped
Line 179 in the Roadmap panel shows ◯ ZIP export as an unfinished item. But exportRealZip() and the full makeZip() implementation were added and wired to the Export Manager button. The roadmap is stale and will mislead users who check it.
Update the roadmap entry to ✓ ZIP export.
